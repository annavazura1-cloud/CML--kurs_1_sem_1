Семантическая разметка

Использование HTML5 тегов для выполнения задания в документах

Что такое семантическая разметка

Использование тегов HTML, которые передают значение и роль своего содержимого, а не только его внешний вид, помогая поисковым системам, браузерам и вспомогательным технологиям лучше понимать структуру и контент веб-страницы. Основные преимущества включают улучшенное SEO, повышающуюся доступность сайта для людей с ограниченными возможностями и лучшую адаптивность для различных устройств.

Структура семантической HTML-разметки

Преимущества семантических тегов

Использование семантических тегов имеет несколько важных преимуществ:

  • Улучшение доступности для пользователей с ограниченными возможностями
  • Повышение SEO-оптимизации за счет лучшего понимания структуры страницы поисковыми системами
  • Упрощение поддержки и чтения кода для разработчиков
  • Обеспечение лучшей совместимости с различными устройствами и браузерами

Основные семантические теги HTML5

HTML5 представил множество новых семантических элементов, включая:

  • <header> - задает верхний заголовочный блок сайта
  • <nav> - определяет навигационные ссылки
  • <main> - задает основное содержимое документа
  • <section> - определяет раздел в документе
  • <article> - задает независимый, самодостаточный контент
  • <aside> - определяет контент, косвенно связанный с основным содержимым
  • <footer> - задает нижний колонтитул страницы или раздела
  • <figure> и <figcaption> - определяют иллюстрации с подписями

Дополнительная информация

Семантическая верстка особенно важна для веб-доступности. Специальные программы (скринридеры) используют семантические теги для навигации по странице и представления контента пользователям с ограниченными возможностями.
